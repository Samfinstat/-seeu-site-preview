# Дополнительные HTML-блоки SeeU

Шаблон персональной страницы поддерживает дополнительные блоки, которые позднее будут добавляться менеджерами из внутренней панели.

## Формат данных

```json
{
  "id": "summer-offer",
  "slot": "after-hero",
  "order": 100,
  "layout": "container",
  "execution": "safe",
  "enabled": true,
  "html": "<div class=\"summer-offer\">...</div>"
}
```

## Доступные позиции

- `after-hero` — после первого экрана;
- `after-catalog` — после услуг или курсов;
- `before-about` — перед блоком о специалисте;
- `after-about` — после блока о специалисте;
- `before-contact` — перед финальным призывом;
- `before-legal` — перед юридическими документами.

## Ширина

- `container` — стандартная ширина страницы;
- `wide` — расширенный блок;
- `full` — на всю ширину экрана.

## Выполнение кода

На первом этапе действует только режим `safe`: HTML и CSS разрешены, а JavaScript, iframe, внешние таблицы стилей, inline-обработчики и действия форм удаляются.

Полный HTML/CSS/JavaScript пока не включается. Он появится только одновременно с внутренней админ-панелью, отдельным правом администратора, журналом изменений, версиями, откатом и Content Security Policy.

Для кнопок, наследующих выбранный клиентом градиент, используется класс `seeu-theme-button`. Внутри блока доступны переменные `--accent`, `--accent-2`, `--accent-soft`, `--ink`, `--muted` и `--line`.

## Требования будущей админ-панели

- разграничение ролей менеджера и администратора;
- предпросмотр до публикации;
- журнал автора, времени и содержимого изменения;
- версии блока и откат;
- публикация сразу на один или выбранные сайты;
- возможность временно отключить блок;
- серверная проверка и Content Security Policy;
- будущий режим JavaScript — только после дополнительного подтверждения администратора и серверной проверки.
